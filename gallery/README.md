# Gallery Photos

This folder holds the haircut photos that appear on the website.

## How it works

1. **Put all haircut gallery photos inside this `gallery` folder.**
2. **Rename them exactly** from `gallery-01.png` through `gallery-15.png`.
3. **Filenames are case-sensitive on GitHub Pages.** `Gallery-01.PNG` will **not** work — use lowercase and the exact names below.
4. **Every file must use the `.png` extension.**
5. **To update a photo, replace the file while keeping its filename.** The website will pick up the new image automatically — no code changes needed.
6. **The first six images** (`gallery-01.png` – `gallery-06.png`) appear in the preview on the homepage.
7. **All fifteen images** (`gallery-01.png` – `gallery-15.png`) appear on the full gallery page (`gallery.html`).

## Photo guidance

- **Recommended dimensions:** roughly **1200 × 1200 px** works well. Most tiles are shown square (1:1); a few taller tiles use a 3:4 ratio, so photos with the subject centered look best.
- **Aspect ratios:** square (1:1) or portrait (3:4) images crop most cleanly. Very wide panoramas will be cropped.
- **Keep file sizes reasonable.** Avoid uploading extremely large images — aim for **under ~400 KB each** where possible.
- **Compress before deploying.** Use a tool like [Squoosh](https://squoosh.app/), TinyPNG, or your phone's export settings to keep the site fast.

## Missing photos

If a photo has not been uploaded yet, the site automatically shows a clean
"Gallery photo coming soon" placeholder in its place — you will never see a
broken-image icon. As soon as you add a correctly named `.png` file, that photo
replaces the placeholder on the next page load.

> Please use real haircut photos from the shop. Do not reuse the general
> storefront or interior photos here.
